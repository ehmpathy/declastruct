# blackbox criteria: declastruct --snap

## usecase.1 = snapshot full state

```
given a wish file with declared resources
  when user runs `declastruct plan --wish resources.ts --into plan.json --snap snapshot.json`
    then snapshot.json is created
      sothat user can inspect full state without re-run
    then snapshot.json contains `observedAt` timestamp
      sothat user knows when the snapshot was taken
    then snapshot.json contains `remote[]` array
      sothat user can see what the API returned
    then snapshot.json contains `wished[]` array
      sothat user can see what they declared
```

## usecase.2 = remote state structure

```
given a resource that exists remotely
  when snapshot is created
    then remote[].forResource.class contains the class name
      sothat user can identify the resource type
    then remote[].forResource.slug contains the unique identifier
      sothat user can identify the specific resource
    then remote[].state contains the full serialized instance
      sothat user sees all fields (readonly included)
    then remote[].state contains `_dobj` with class name
      sothat user can identify the domain object type
```

## usecase.3 = wished state structure

```
given a resource declared in the wish file
  when snapshot is created
    then wished[].forResource.class contains the class name
    then wished[].forResource.slug contains the unique identifier
    then wished[].state contains the full serialized instance
    then wished[].state contains `_dobj` with class name
```

## usecase.4 = new resource (no remote state)

```
given a resource declared in wish but not yet created remotely
  when snapshot is created
    then remote[] contains an entry for the resource
    then remote[].forResource identifies the resource
    then remote[].state is null
      sothat user knows the resource doesn't exist remotely yet
    then wished[] contains the declared state
```

## usecase.5 = snapshot scope matches plan

```
given resources declared in wish file
  when snapshot is created
    then only resources in the wish are included
      sothat snapshot scope matches plan.json scope
    then resources not in wish are excluded
      sothat user sees only what they asked for
```

## usecase.6 = opt-in behavior

```
given user runs plan without --snap flag
  when plan completes
    then no snapshot file is created
      sothat user only gets snapshot when they ask for it

given user runs plan with --snap flag
  when plan completes
    then snapshot file is created at specified path
```

## usecase.7 = snapshot independent of plan outcome

```
given resources with no changes needed (action = KEEP)
  when snapshot is created
    then full remote and wished state is captured
      sothat user can audit even when no changes

given resources with changes needed (action = CREATE, UPDATE, DESTROY)
  when snapshot is created
    then full remote and wished state is captured
      sothat user can debug the difference
```
