# blueprint: arg passthrough

> **root motto:** don't let declastruct interfere with --arg passthrough

---

## summary

implement process.argv injection for passthrough args:
1. user passes args after `--` separator (commander captures as positional args)
2. plan command injects captured args into process.argv before wish import
3. wish files parse args via standard node patterns (parseArgs, etc.)
4. apply command ignores passthrough args (uses plan's captured state)

**benefit:** `--` requirement means declastruct catches typos like `--wishe` instead of silent passthrough.

## implementation details

### invoke.ts changes

```typescript
const planCommand = program
  .command('plan')
  .description('Generate a change plan from a wish file')
  .requiredOption('--wish <file>', 'Path to wish file')
  .requiredOption('--into <file>', 'Path to output plan file')
  .usage('--wish <file> --into <file> [-- <wish-args>]')  // <-- ADD: help text
  .action(async (options, command) => {
    // capture args after -- separator
    const passthroughArgs = command.args;  // <-- ADD
    await executePlanCommand({
      wishFilePath: options.wish,
      planFilePath: options.into,
      passthroughArgs,  // <-- ADD
    });
  });

// <-- ADD: guide user on unknown options
planCommand.on('option:unknown', (flag) => {
  console.error(`error: unknown option '${flag}'`);
  console.error(`hint: to pass args to your wish file, use: -- ${flag}`);
  process.exit(1);
});
```

### wish-with-args.fixture.ts

```typescript
import { parseArgs } from 'util';
import type { DomainEntity } from 'domain-objects';
import type { DeclastructProvider } from '@src/domain.objects/DeclastructProvider';
import { DemoResource, demoProvider, genSampleDemoResource } from './providers/demo.provider';

export const getResources = async (): Promise<DomainEntity<any>[]> => {
  // parse args inside function so each call sees current process.argv
  // (avoids module cache issues when imported multiple times with different args)
  const { values } = parseArgs({
    args: process.argv.slice(2),
    options: {
      env: { type: 'string', default: 'test' },
    },
    strict: false,  // allow unknown flags
  });

  const suffix = values.env === 'prod' ? '-production' : '-test';
  return [
    genSampleDemoResource({ name: `Resource${suffix}` }),
  ];
};

export const getProviders = async (): Promise<DeclastructProvider<any, any>[]> => {
  return [demoProvider];
};
```

### readme.md changes

add section documenting wish arg passthrough:

```markdown
## wish arg passthrough

pass args to your wish file via `--` separator:

\`\`\`bash
npx declastruct plan --wish resources.ts --into plan.json -- --env prod --verbose
\`\`\`

args after `--` are injected into `process.argv` before the wish file loads. parse them with node's `parseArgs`:

\`\`\`typescript
import { parseArgs } from 'util';

const { values } = parseArgs({
  args: process.argv.slice(2),
  options: {
    env: { type: 'string', default: 'test' },
    verbose: { type: 'boolean', default: false },
  },
});

export const getResources = async () => {
  // use values.env, values.verbose to configure resources
};
\`\`\`

**typo safety:** unknown flags without `--` trigger an error with guidance:

\`\`\`
error: unknown option '--env'
hint: to pass args to your wish file, use: -- --env
\`\`\`
```
