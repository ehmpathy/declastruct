# research: prod codepaths for arg passthrough

---

## pattern.1: CLI entry point (invoke.ts)

**location:** `src/contract/cli/invoke.ts`

**current behavior:**
- uses Commander.js for CLI argument parse
- defines plan and apply commands with required/optional flags
- does NOT use `allowUnknownOption()` — unknown flags cause errors

**citations:**

[1] commander setup
```typescript
const program = new Command();

program
  .name('declastruct')
  .description('Declarative resource structure control')
  .version(pkg.version);
```

[2] plan command definition
```typescript
program
  .command('plan')
  .description('Generate a change plan from a wish file')
  .requiredOption('--wish <file>', 'Path to wish file')
  .requiredOption('--into <file>', 'Path to output plan file')
  .action(async (options) => {
    // ...
  });
```

[3] parse invocation
```typescript
await program.parseAsync(args, { from: 'user' });
```

**relation to wish:**
- [EXTEND] must add `allowUnknownOption()` to capture unknown flags
- [EXTEND] must capture unknown args and pass to plan command
- [EXTEND] help text should show `[-- <wish-args>]`

---

## pattern.2: plan command (plan.ts)

**location:** `src/contract/cli/plan.ts`

**current behavior:**
- resolves wish file path
- dynamically imports wish file via `await import()`
- calls `getResources()` and `getProviders()` with NO arguments

**citations:**

[4] wish file import
```typescript
// import wish file
const wish = await import(resolvedWishPath);
```

[5] export calls with no arguments
```typescript
// get resources and providers
const resources: DeclaredResource[] = await wish.getResources();
const providers = await wish.getProviders();
```

**relation to wish:**
- [EXTEND] must inject passthrough args into process.argv BEFORE import
- [REUSE] the dynamic import pattern stays the same
- [REUSE] the export calls stay the same (no function parameters needed)

---

## pattern.3: apply command (apply.ts)

**location:** `src/contract/cli/apply.ts`

**current behavior:**
- loads plan from file (standard mode) or uses wish directly (yolo mode)
- imports wish file and calls exports with NO arguments
- same pattern as plan.ts

**citations:**

[6] wish file import in apply
```typescript
// import wish file
const wish = await import(resolvedWishPath);
```

[7] export calls with no arguments
```typescript
// get resources and providers
const resources: DeclaredResource[] = await wish.getResources();
const providers = await wish.getProviders();
```

**relation to wish:**
- [REUSE] apply ignores passthrough args by design (plan/apply contract)
- [REUSE] no changes needed to apply command behavior

---

## pattern.4: process.argv state

**location:** global node.js state

**current behavior:**
- process.argv contains full command line at startup
- wish files could already parse it, but it contains declastruct's args too
- no mechanism to filter declastruct args from wish args

**relation to wish:**
- [EXTEND] plan command must replace process.argv with only passthrough args
- [EXTEND] must preserve original argv for restore after import (optional safety)

---

| plan.ts | EXTEND | inject passthrough args into process.argv before import |
| apply.ts | REUSE | no changes — ignores passthrough args |
